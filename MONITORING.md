# Система мониторинга рекомендаций

## 1. Технические метрики (Infrastructure)
Отслеживаются через Middleware FastAPI и логируются в `logs/infrastructure.log`.

| Метрика | Описание | Порог алерта |
|---------|----------|--------------|
| `request_latency_p95` | 95-й перцентиль времени ответа API | > 500ms |
| `error_rate_5xx` | Доля серверных ошибок | > 1% |
| `memory_usage` | Потребление RAM процессом | > 80% |

## 2. Метрики качества модели (Model Performance)
Рассчитываются пайплайном Airflow после переобучения и логируются в `logs/model_metrics.jsonl`.

- **Recall@10**: Доля товаров из корзины, попавших в топ-10 рекомендаций;
- **Precision@10**: Точность рекомендаций;
- **Coverage**: % товаров каталога, участвующих в рекомендациях.

*Логика использования:* Если `Recall@10` падает более чем на 10% относительно baseline, срабатывает алерт в Telegram, и пайплайн откатывается на предыдущую версию модели.

## 3. Data Drift
- **PSI (Population Stability Index)** для признаков `age`, `renta`;
- **Drift в таргетах**: Мониторинг конверсии в корзину по дням.

## 4. Интеграция
Метрики отправляются:
1. В JSON-логи (локально);
2. В Telegram-бот (через webhook при критических отклонениях).

# mle-banking-recsys-project-2026

## Описание проекта
Система рекомендаций банковских продуктов для кросс-продаж. Задача: на основе исторических данных о клиентах предсказать топ-K продуктов, которые клиент с наибольшей вероятностью приобретёт в следующем периоде.

### Бизнес-цель
Увеличение конверсии кросс-продаж при контроле маркетинговых затрат и рисков.

### Технологический стек
- Python 3.10, pandas, scikit-learn, lightgbm/catboost
- MLflow для логирования экспериментов и версионирования моделей
- FastAPI для REST API, Docker для контейнеризации
- Airflow для оркестрации переобучения (опционально)

## Руководство по проекту

### Трансляция бизнес-задачи в техническую
- Бизнес-задача: максимизация количества успешных кросс-продаж при ограничении на бюджет рекомендаций.
- Техническая задача: multilabel-классификация с ранжированием продуктов по вероятности приобретения.
- Метрики:
    - Технические: Precision@K, Recall@K, MAP@K, ROC-AUC per label.
    - Бизнес-интерпретация: Precision@K снижает стоимость ложных рекомендаций, Recall@K минимизирует упущенную выручку, MAP@K оптимизирует порядок вывода в интерфейсе.

### Разворачивание инфраструктуры обучения
- Запуск MLflow: bash scripts/setup_mlflow.sh
- Скрипт инициализирует tracking server с файловым бэкендом и артефакт-хранилищем в ./mlruns.
- Все эксперименты логируются с параметрами, метриками, артефактами и версиями кода.

### Проведение EDA
- Анализ временной динамики: распределение записей по fecha_dato, выявление сезонности.
- Анализ таргетных колонок (ind_*_ult1): частота приобретения, дисбаланс классов.
- Сегментация клиентов по age, renta, antiguedad, segmento.
- Выводы: обработка пропусков в renta, кодирование категориальных признаков, выбор стратегии валидации по времени.

### Генерация признаков и обучение модели
- Признаки: лаги активности, агрегации по клиенту, временные фичи, взаимодействия категориальных переменных.
- Пайплайн: sklearn ColumnTransformer с импутерами и энкодерами, фиксация random_state.
- Эксперименты в MLflow: сравнение LogisticRegression, RandomForest, LightGBM.
- В логах: параметры, метрики на валидации, feature importance, конфиги.

### Разворачивание инфраструктуры применения
- REST API на FastAPI: эндпоинт /predict принимает список клиентов, возвращает топ-K продуктов.
- Загрузка модели из MLflow Model Registry.
- Контейнеризация: docker build -t bank-recsys ., запуск через docker-compose.
- Скрипт запуска: bash scripts/setup_production.sh поднимает API, мониторинг и scheduler.

### Мониторинг
- Технические метрики: latency P95, throughput, ошибки 4xx/5xx.
- Качество: Precision@K на отложенной выборке, распределение предсказаний.
- Data drift: PSI для ключевых признаков, мониторинг сдвига таргета через симуляцию ground truth.
- Бизнес-метрики: конверсия рекомендаций, отток после рекомендаций.
- Метрики отправляются через JSON-логирование или Prometheus client (опционально).
